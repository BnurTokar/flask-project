Docker

You are getting the basic knowledge about containers, images, docker hub and you are going to learn how to start a project and access the container with the terminal using terminal commands.


Containers:

Containerization and virtualization look similar but they are for different purposes. Container engine provides Apps Operating System (OS).

In Virtual Machines, all computer have their own RAM. In Containers, infrastructure has RAM. Operating system have access to the RAM.

Images:

A Docker image is a read-only template that contains a set of instructions for creating a container that can run on the Docker platform.

The Docker Command:

We are using the command line because web servers have a composition.

“ docker ps ”

- See if anything already start it up.

“docker kill -image number-”

- Stop running images.

“docker compose up –build”

- Build an image.

“ docker run -itp 80:80 ubuntu “

- Run the image in port 80 of the network on your computer.

How to Start Project:

You can start the project using build an image command in the terminal. And you can access the container by using “docker ps” command in the terminal.
