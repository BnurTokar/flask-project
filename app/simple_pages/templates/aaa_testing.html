AAA Testing

Arrange-Act-Assert

Arrange-Act-Assert (AAA) is a pattern for writing good tests. In the arrange step, the test concept sets up inputs and targets for the act step. Then, the act block calls methods under test along with the arranged parameters. Finally, the assert step expects an outcome or response, and determines the goodness of the action of the test method.

There are different test frameworks in Python such as Robot Framework, Nose2, Pytest. This tutorial provides an example by using Pytest.

Why Is Testing Important ?

Testing provides so many advantages for both project and developer perspectives.

Primarily, it helps developers to recognize possible errors and warnings before deploying into production. On the other hand, it speeds up the process and decreases the cost of the project and protects the codebase since having a hanging code in a large project can be quite expensive which causes damages and possibile breaking to the codebase. Moreover, the debugging process is going to be simple as the developer is going to recognize bugs or any problems in the system in the early stage of the software development life cycle. Also it improves readability of the code by reducing the number of bugs occured in the code.

To sum up, testing is important because it improves the code readability, speeds up the process, decreases the cost of the project, protects the codebase and simplifies the debugging process.
